🔍 什麼是運算思維？

運算思維不只屬於資訊人，而是每個人都能學習、應用的解題方式。

📖 學術定義

"運算思維是利用電腦科學的基本概念進行問題解決、系統設計與人類行為理解的思維模式。"
— Wing, J. M. (2006). Communications of the ACM, 49(3), 33-35.
"運算思維讓我們能擁有電腦科學家面對問題時所持有的思維模式。"
— Grover & Pea (2013). Educational Researcher, 42(1), 38–43.

🌐 Google 的定義

Computational Thinking (CT) is a problem solving process.

CT 不只用於開發程式，也可應用於人文、數學、科學等各領域的問題解決。

💼 為什麼非資訊科系也需要運算思維？

  • 科學與工程：利用運算模擬建築結構安全性、預測氣象
  • 人文與社會：分析廣告投放策略、人口老化與醫療資源分布
  • 藝術領域：構建三維動畫、創作數位音樂

🔢 邏輯運算與布林值

電腦只認識 True (1) 和 False (0)，所有判斷都由三種運算組成：

AND（且）

兩個條件都要 True，結果才是 True

OR（或）

只要一個條件為 True，結果就是 True

NOT（非）

反轉真假：True 變 False，反之亦然

🧩 物件導向程式設計 (OOP)

整個系統由一個個物件 (Object) 組成，每個物件都有：

  • 📦 屬性 (Attribute)：靜態的特徵（顏色、大小、位置）
  • ⚡ 方法 (Method)：物件可執行的動作（移動、旋轉、說話）

設計步驟：① 確認需要哪些物件 → ② 定義屬性與方法 → ③ 設計演算法

🔁 流程控制結構

  • 📌 循序結構 (Sequences)▼
    按照順序一步一步執行。
    範例：開機 → 輸入密碼 → 開啟應用程式
  • 🔄 重複結構 (Loops)▼
    讓同一段程式重複執行，直到條件不成立。
    範例：重複播放音樂，直到使用者按下停止
  • 🤔 條件判斷 (Conditionals)▼
    根據條件決定執行哪一段程式。
    範例：如果分數 ≥ 60 → 顯示「及格」，否則「不及格」
  • ⚡ 事件驅動 (Events)▼
    某個事件發生時才觸發程式執行。
    範例：當使用者按下按鈕 → 播放動畫
  • ⏸️ 平行結構 (Parallelism)▼
    讓多個程式同時執行。
    範例：遊戲中角色移動的同時，背景音樂也在播放
